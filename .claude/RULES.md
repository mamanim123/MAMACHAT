## 핵심 운영 규칙

### 작업 관리 규칙
- TodoRead() → TodoWrite(3개 이상 작업) → 실행 → 진행 상황 추적
- 가능한 경우 배치 도구 호출 사용, 종속성이 있을 때만 순차적
- 실행 전에 항상 유효성 검사, 완료 후에 확인
- 작업을 완료로 표시하기 전에 린트/타입 체크 실행
- 복잡한 다중 세션 워크플로우에는 /spawn 및 /task 사용
- 작업 전반에 걸쳐 90% 이상의 컨텍스트 유지

### 파일 작업 보안
- 쓰기 또는 편집 작업 전에 항상 읽기 도구 사용
- 절대 경로만 사용, 경로 탐색 공격 방지
- 배치 작업 및 트랜잭션과 유사한 동작 선호
- 명시적으로 요청하지 않는 한 자동으로 커밋하지 않음

### 프레임워크 준수
- 라이브러리 사용 전에 package.json/pyproject.toml 확인
- 기존 프로젝트 패턴 및 규칙 따르기
- 프로젝트의 기존 가져오기 스타일 및 구성 사용
- 프레임워크 수명 주기 및 모범 사례 존중

### 체계적인 코드베이스 변경
- **필수**: 변경 전에 프로젝트 전체 검색 완료
- 대상 용어의 모든 변형에 대해 모든 파일 유형 검색
- 컨텍스트 및 영향 평가와 함께 모든 참조 문서화
- 종속성 및 관계에 기반한 업데이트 순서 계획
- 계획에 따라 조정된 방식으로 변경 실행
- 포괄적인 변경 후 검색으로 완료 확인
- 관련 기능이 계속 작동하는지 유효성 검사
- 범위가 불확실할 때 포괄적인 검색을 위해 Task 도구 사용
